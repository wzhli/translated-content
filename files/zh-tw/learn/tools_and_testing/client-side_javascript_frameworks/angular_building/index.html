這篇文章內容涵蓋，如何建立一個生產版本（production）的應用程式，以及提供後續的學習資源。

預備知識： 熟悉 HTML、CSS 和 JavaScript 的主要概念，以及 終端機／命令列 的知識。
學習目標： 學會如何編譯你的 Angular 應用程式

建置最終版的應用程式

現在你已經完成開發你的應用程式，接著執行 Angular CLI build 的指令。 當你在 todo 的目錄底下執行 build 的指令，應用程式會編譯並且輸出到 dist/ 的目錄下。

在 todo 目錄底下，且在命令列執行以下的命令：

ng build --prod

命令列介面—— CLI（Command Line Interface），會將應用程式編譯並且輸出到新的目錄 dist 底下。 這個 --prod 尾隨在 ng build 的參數，會移除在正式產品版本上不需要的東西。

部署你的應用程式

為了部署你的應用程式，你可以複製 dist/my-project-name 資料夾底下的內容到你的伺服器上。 因為這些都是靜態檔案，你可以將這些檔案放到任何能夠提供檔案的伺服器上，像是：

  • Node.js
  • Java
  • .NET

你可以使用任何後端，像是 Firebase、Google Cloud 或 App Engine。

接著要做什麼

現在你已經建立了基本的應用程式，但是你的Angular學習旅程才剛開始，你可以探索Angular文件學習更多，像是：

  • 英雄之旅(正體中文版)：一個深入的 Angular 重點課程，像是使用服務（service）、導覽（navigation）以及從伺服器取得資料。
  • Angular 元件 指南：一系列的文章，主題包括生命週期、元件互動以及檢視封裝。
  • 表單 指南：文章內容包含建立Angular的響應式表單、輸入驗證以及建立動態表單。
